### Bug #2 Fix: RLS Policy for Submissions (Migration)

**Problem:** The current UPDATE policy `Students can update ungraded submissions` uses `marks IS NULL` which is correct. However, the SubmitAssignment.tsx code does not verify the update actually succeeded (no row count check).

**Database change:** The RLS policy already uses `marks IS NULL` (confirmed from live DB). No migration needed for the policy itself.

**Code change in `SubmitAssignment.tsx`:** Add `.select('id')` after update to detect silent RLS rejections:

```text
After .update(baseSubmission).eq('id', existingSubmission.id), add .select('id')
Check if returned data is empty - throw meaningful error about graded submissions
```

---

### Bug #4 Fix: Create `resolve-submission-files` Edge Function

**Problem:** No edge function exists for securely resolving signed URLs. The `FilePreviewDialog` currently shows raw public URLs from `uploads` bucket (which is private), so images fail to load.

**New file:** `supabase/functions/resolve-submission-files/index.ts`

- Extract Bearer token from request
- Verify user identity via `supabaseAdmin.auth.getUser(token)`
- Look up the submission by ID
- Check authorization: user is the student owner, a faculty member for the assignment, or an admin
- Generate signed URLs for all paths in `file_urls` (and `file_url` fallback)
- Return the signed URLs array

**Config update:** Add `[functions.resolve-submission-files]` with `verify_jwt = false` to `supabase/config.toml`

### Bug #1 Fix: CDN Cache-Busting for Reference Image

**File:** `supabase/functions/verify-handwriting/index.ts`

### Bug #3 Fix: Mobile Gallery Upload

**File:** `src/pages/student/SubmitAssignment.tsx`

**Changes:**

1. **Update `validateImageFile**` to include file extension fallback:
  - Check both `file.type` AND file extension
  - Accept `.jpg`, `.jpeg`, `.png`, `.webp` extensions even when `file.type` is empty (Android gallery issue)
2. **Add `normalizeImageFile` function** before upload:
  - Resize images to max 2048px dimension
  - Convert to JPEG at 0.85 quality
  - Release canvas memory explicitly after conversion
  - This prevents mobile OOM crashes from 12MP camera photos
3. **Update file input element:**
  - Add explicit MIME types: `accept="image/jpeg,image/png,image/webp,.jpg,.jpeg,.png,.webp"`
  - Do NOT add `capture="environment"` (breaks gallery access on iOS)
4. **Update upload logic:** Pass normalized blob with explicit `contentType: 'image/jpeg'` and `cacheControl: 'no-cache'`
